# vokra (Python binding)

Python binding for **Vokra**, an audio-specialized inference runtime
(TTS / ASR / VAD / Speaker-ID). This package is a thin
[`ctypes`](https://docs.python.org/3/library/ctypes.html) wrapper over the
Vokra C ABI (`include/vokra.h`); the actual inference runs in a prebuilt
native library (`libvokra.dylib` / `libvokra.so` / `vokra.dll`) that is
bundled inside the wheel.

## Status: pre-1.0 (ABI unfrozen — IF-01)

The Vokra C ABI is **not frozen** before v1.0 (see ADR-0003, requirement
IF-01). Every pre-1.0 release may change the C ABI, and therefore the
Python surface, without deprecation windows. Consequences:

- **Pin an exact version.** Do not use `~=` or `>=` ranges until v1.0.

  ```
  pip install "vokra==0.1.0"
  ```

- **Upgrading across minor versions may require code changes.**
- **`vokra.__abi_version__`** is exposed and must match the loaded native
  library at import time; a mismatch raises `VokraError` immediately.
- The 14 exported C symbols in `include/vokra.h` are the contract; the
  Python surface tracks them 1:1 and is regenerated by
  `scripts/gen-py-bindings.py`.

## Design invariants

- **Zero third-party runtime dependencies (NFR-DS-02, "zero-dep").**
  Installing `vokra` never pulls a Python package from PyPI other than
  `vokra` itself. `numpy` is an optional interop, guarded by a lazy import
  in `audio.py`. Root `Cargo.lock` is unaffected — this binding adds no
  Rust code and no new crate.
- **Explicit errors, no silent fallback (FR-EX-08).** If a GPU backend is
  unavailable, an operator is unsupported, or a model file is malformed,
  the wrapper raises a typed `VokraError` subclass. The runtime never
  silently falls back to CPU. See "Error contract" below.
- **Apache-2.0.** No GPL / LGPL code, direct or transitive (NFR-LC-02).
  Enforced in CI by `cargo deny check licenses` (native side) and
  `pip-licenses --allow-only 'Apache-2.0;MIT;BSD-3-Clause;BSD-2-Clause;PSF-2.0;Python-2.0'`
  (Python side).
- **Locale-independent (NFR-RL-01).** No `strtod`; ctypes marshaling
  (`c_float`, `c_int32`, `c_size_t`) is used throughout. Tested under
  `LC_ALL=de_DE.UTF-8` in CI.

## Error contract

Every C ABI call is checked against `vokra_status_t` in the same Python
call frame; `vokra_last_error()` is read immediately on any non-OK status
(`vokra_last_error()` is thread-local — reading it from a different
thread returns NULL). The 10 status codes map 1:1 to exception classes:

| `vokra_status_t`               | Python exception            |
| ------------------------------ | --------------------------- |
| `VOKRA_OK`                     | (no exception)              |
| `VOKRA_ERROR_INVALID_ARGUMENT` | `VokraInvalidArgument`      |
| `VOKRA_ERROR_IO`               | `VokraIoError`              |
| `VOKRA_ERROR_MODEL`            | `VokraModelError`           |
| `VOKRA_ERROR_UNSUPPORTED_BACKEND` | `VokraUnsupportedBackend` |
| `VOKRA_ERROR_ALLOC`            | `VokraAllocError`           |
| `VOKRA_ERROR_DECODE`           | `VokraDecodeError`          |
| `VOKRA_ERROR_STREAM`           | `VokraStreamError`          |
| `VOKRA_ERROR_PANIC`            | `VokraPanic`                |
| `VOKRA_ERROR_OTHER`            | `VokraOtherError`           |

All inherit from `VokraError`, so `except VokraError` catches everything.

## Install

```
pip install "vokra==0.1.0"
```

The wheel bundles the native library for your platform; no separate
install step is needed. CUDA / Metal are auto-detected inside the native
runtime and used only when the model was built for them — otherwise the
call raises `VokraUnsupportedBackend` (no silent fallback).

## Platform support (initial)

| OS       | Arch                       | Wheel tag                     |
| -------- | -------------------------- | ----------------------------- |
| Linux    | x86_64                     | `manylinux_2_28_x86_64`       |
| Linux    | aarch64                    | `manylinux_2_28_aarch64`      |
| macOS    | universal2 (x86_64 + arm64)| `macosx_11_0_universal2`      |
| Windows  | x86_64                     | `win_amd64`                   |

Python 3.9 – 3.12 supported. iOS / Android / WASM are out of scope for
the Python binding — use the Swift / Kotlin bindings (planned) or the
WASM target (v1.5+) instead.

## Build (from source)

```
cd bindings/python
pip wheel . -w dist/ --no-deps
```

The wheel is only usable if the prebuilt native library has been placed
in `src/vokra/_lib/`. In CI this is done by `python-wheel-build` in
`.github/workflows/ci.yml`, which downloads the `vokra-capi-*` artifact
from the Rust build job. For local builds, copy the library manually:

```
cp target/release/libvokra.dylib src/vokra/_lib/     # macOS
cp target/release/libvokra.so    src/vokra/_lib/     # Linux
copy target\release\vokra.dll     src\vokra\_lib\    # Windows
```

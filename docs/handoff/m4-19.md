## What landed (completion gaps closed)

M4-19 completes the M2-09 Wyoming server (it was **not** a new implementation —
`docs/m4-scope-expansion-2026-07-13.md` §BIG-9 事実訂正 2). Gaps closed:

1. **accept-loop completion (T02)** — `server.rs::wyoming_accept_loop` is now
   service-aware: `run_wyoming_connection` (real ASR+TTS) when a
   `WyomingBackend` registry is wired, `run_describe_only_connection` otherwise.
   `spawn_server` delegates to `spawn_server_with_service(.., None, None)`, so
   the production startup path stays discovery-only and regression-free.
2. **synthesize dispatch (T03)** — one connection handler routes `synthesize`
   → `handle_synthesize` (was `other => unsupported`); `transcribe` runs on
   `spawn_blocking`. A reader-pump child task frames every event onto a bounded
   mpsc (R5, no read cancel-safety hazard).
3. **multi-session (T04)** — each connection takes an M3-15 `Scheduler`
   permit + `StreamSlot` (RAII release); overload → explicit `error` event via
   `SchedulerError::to_server_error` (FR-EX-08).
4. **barge-in (T05/T06)** — `wyoming::BargeIn` (an `Arc<AtomicBool>` mirroring
   M3-14 `InterruptHandle`) is polled at each `audio-chunk` boundary; a mid-TTS
   `audio-start` raises it → remaining chunks dropped + immediate `audio-stop`.

**Tests** (all green): vokra-server lib +11 (34 in `api::wyoming`), integration

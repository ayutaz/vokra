## (b) The honest feeder constraint (do NOT read this as "WFST ASR works e2e")

`wfst_decode` consumes a per-frame acoustic **emission** matrix (a CTC / RNN-T posterior). But `ctc_decode` / `rnnt_decode` (FR-OP-41/42) are **reserved and unimplemented** (`crates/vokra-core/src/m5_residual_ops.rs`, "NeMo-family trigger pending"), and Vokra's only live acoustic decoder — attention `beam_search` (Whisper/Voxtral) — returns *token sequences*, not an emission matrix. Therefore M5-06 lands `wfst_decode` as a **decode-only** primitive:

- **verified** against offline reference emissions (the parity fixtures), and
- **usable** for lexicon / grammar rescoring at the word level (the FR-OP-43 domain-adaptation use), but
- a **classic HCLG frame-synchronous e2e ASR** waits on the CTC/RNN-T feeder landing.

The API does not pretend a live ASR path exists.

## (c) C consumers cannot call `wfst_decode` during the rc window

Per ADR M5-06, the C-surface decision is deferred to the M5-13 freeze — `include/vokra.h` gains **no** `wfst_*` symbols in M5-06 (`gen-c-abi.sh --check` = no diff). During v1.0-rc the only surface is the Rust API `vokra_core::decode::wfst::*`. Whether to export a C entry point (and its exact shape) is an M5-13 freeze decision.

## (e) Deferred (blocked on the CTC/RNN-T feeder or owner corpora — not implementation gaps)

1. **Live e2e HCLG ASR** — needs FR-OP-41/42 (reserved). The decoder + parity are complete; only the *live feeder* is missing.
2. **beam_search n-best → domain-FST rescoring wiring** — the ADR proposes deferring the live model wiring until the feeder lands (rescoring API surface is available; real-model wiring is not exercised without a feeder).
3. **`log` semiring** — future additive (posteriors / forward-backward); the `Semiring` trait is carved so it lands without reshaping the decoder.
4. **General FST algebra** (`compose`/`determinize`/`minimize`) — intentionally out of scope; HCLG is composed offline by the developer-side OpenFST toolchain.
5. **Real domain HCLG / lexicon demo** (medical / legal corpora) — owner-supplied artefacts with their own license judgement.

## (f) Regenerating the fixtures

Requires real OpenFST (dev-time only, **not** a runtime dep):

```
brew install openfst          # Apache-2.0, bottled
python3 tools/parity/wfst_dump_reference.py
```

The script **aborts loudly** if the OpenFST CLI is absent — it never falls back to a self-computed reference (a self-mirror verifies nothing; numerical-parity discipline). It regenerates `crates/vokra-core/tests/parity/fixtures/m5-06/{linear,epsilon,nbest}.{fst,expected}`.

> **Oracle note (ADR §未確定 #4 resolution)**: the reference uses the OpenFST **CLI tools** (`fstcompile`/`fstcompose`/`fstshortestpath`), not `pywrapfst` — no installable `pywrapfst`/`openfst_python`/`pynini` wheel exists for Python 3.9 / macOS-arm64 (probed 2026-07-20). The CLI is the same Apache-2.0 OpenFST codebase and is an equally independent oracle.

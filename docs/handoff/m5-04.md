# M5-04 handoff: console-portability static-link base + `check-console-static.sh` gate

- **Producer**: M5-04 (CC), branch `feat/m5-plan-and-wave1`.
- **Requirements**: FR-API-01 (C ABI = the console porting base), FR-API-03 (thread safety), NFR-RL-03 (static-only distribution, the iOS precedent), NFR-RL-05 (JIT-free), NFR-DS-02 (zero-dep), FR-EX-08 (explicit errors, no silent fallback), IF-01 / M5-13 (C ABI freeze — this WP adds **no** C symbol).
- **Spec**: `docs/tickets/m5/M5-04-console-portability-static-link.md` (gitignore-local). **ADR**: `docs/adr/M5-04-console-portability.md` (gitignore-local, **Proposed** — owner ratifies the console-specific parts after the SDK arrives).
- **Boundary in one line**: a console is a *no-dynamic-loading* environment; the console SDK, its NDA, and per-target distribution are **owner** territory. CC lands the static-link **base** and the machine gate that proves it is sound, exactly as M2-02 landed the iOS static-only base and left device distribution to the owner.

## (a) What landed (CC-side)

`scripts/check-console-static.sh` — a zero-dep gate (bash + cargo + nm only; no python, no crate, no committed binary). Three legs + a `--self-test`:

| Leg | What it asserts |
|-----|-----------------|
| C ABI completeness | Every function symbol declared in `include/vokra.h` (extracted **at runtime**, not hardcoded) is DEFINED in the CPU-only static archive. Direction is header → archive, so a declared-but-unexported symbol (an ABI regression) is caught. |
| FFI panic firewall | `[profile.release]` does **not** carry `panic = "abort"` (and RUSTFLAGS has no `panic` override). `crates/vokra-capi/src/ffi_guard.rs`'s `catch_unwind` becomes a no-op under `abort`; a panic would then abort the whole host process. `[profile.release-min]` (non-FFI standalone bins) may still use abort — the check discriminates the two sections and strips comments before matching. |
| No-dynamic-load | The CPU-only archive references no `dlopen`/`dlsym`/`dlvsym`/`LoadLibrary(A/W)`/`GetProcAddress` — the literal meaning of a static-link build. |

Usage:

```
scripts/check-console-static.sh              # build CPU-only staticlib for $VOKRA_STATIC_TRIPLE + verify
scripts/check-console-static.sh <path.a>     # verify an already-built archive
scripts/check-console-static.sh --self-test  # negative-fixture suite (hermetic, no build)
```

Design points worth knowing (all measured on the M1 iMac, 2026-07-21):

- **Default target `x86_64-unknown-linux-musl`** — a fully-static, stock-rustup proxy for a no-dynamic-load environment, already CI-built in the `server-deployment` job. It is also the only *reliable* target for symbol enumeration on macOS: a host fat-LTO build leaves `vokra-capi`'s codegen object as LLVM bitcode the system `nm` cannot read, so the 33 C ABI symbols would be invisible; a cross build (musl, and any real console triple) emits native objects `nm` reads. The owner injects the real target via `VOKRA_STATIC_TRIPLE`.
- **nm is chosen by capability probe** (`$VOKRA_NM` → `$LLVM_NM` → `nm` → `llvm-nm`), not by binary name, and fails loud only if nothing can read the archive. An SDK-supplied `nm` can be pointed to via `VOKRA_NM` / `LLVM_NM`.
- **No `nm -u | grep -q`** — the variable-capture idiom is used throughout to avoid the SIGPIPE-into-pipefail fail-open the iOS verifier documents.
- **No release-profile change, no new C symbol** — `include/vokra.h` is unchanged, `gen-c-abi.sh --check` drift is zero, root `Cargo.lock` is unchanged.

## (b) Verify (M1 iMac, 2026-07-21)

- `scripts/check-console-static.sh --self-test` = **10 passed, 0 failed** (each leg proven to detect its failure: panic-inject → red, missing symbol → red, dlopen ref → red, plus the green controls: commented panic, `release-min` abort, benign `my_dlopen_wrapper`, non-object nm-probe rejection).
- `scripts/check-console-static.sh` (no-arg, builds CPU-only musl staticlib) = **all checks passed**: all 33 header symbols defined, 0 dlopen refs.
- Adversarial (proves the gate really goes red on real archives): a **GPU-feature** static build has the 33 symbols present **and** references `dlopen`/`dlsym` → the no-dynamic-load leg fails as designed; a non-ar file fails the file leg; a dlopen-only crate archive fails the C ABI leg (0/33). This is the direct demonstration that a GPU backend re-introduces dynamic loading while the CPU-only build does not.
- `python3 scripts/compliance/lint-pipefail-grep-q.py scripts/check-console-static.sh` = OK. `shellcheck -S warning` = clean. `git diff include/ Cargo.lock` = empty.

## (c) Owner tasks (queue — do not shrink)

1. **Console platform NDA** (依頼者). The static-link SDK is only obtainable under NDA from the console platform holders — CC cannot substitute this. It is the critical-path prerequisite for everything below.
2. **SDK → real target build + verify** (依頼者). With the SDK toolchain installed:
   `VOKRA_STATIC_TRIPLE=<the SDK's target triple> scripts/check-console-static.sh`
   then `scripts/check-console-static.sh <the produced .a>`, using the CC musl proxy as the reference. **The real target triple is passed via the environment variable and must not be written into any tracked file** (handoff / CI YAML / platform-support). If it must be recorded, put it in a gitignore-local internal note.
   - If the SDK ships its own `nm`, point the probe at it with `VOKRA_NM=<sdk-nm>`.
   - If the SDK toolchain rejects unwinding and demands `panic = "abort"`: **do not** set it on `[profile.release]` (that silently defeats the C ABI panic firewall). Raise it as a design question — the correct move is a dedicated non-FFI profile, not weakening the shipped runtime.
3. **ADR ratification** (依頼者). Mark `docs/adr/M5-04-console-portability.md` **Accepted** once the console-specific parts (target triple family, SDK `nm`, distribution shell) are confirmed against the real SDK. CC left it Proposed.
4. **CI wiring** (依頼者, optional). If desired, add a `workflow_dispatch` job mirroring `gpu-vulkan-parity.yml`'s posture (not a required check; first run owner-triggered). Not wired by CC because the useful run needs the real SDK target, which is owner-only.

## (d) Deferred (additive; not implementation gaps — none blocks the base being sound)

The full M5-04 ticket also lists a distribution "shell" and auxiliary tooling that this wave intentionally did not build, matching the launching scope (gate + ADR + handoff):

1. `dist/console-static/` package layout + NOTICE variant + SBOM wiring (NFR-LC-01 / NFR-MT-03) — the M4-15 build-target 4-piece set, adaptable here; owner assembles the real-triple artifact into the same shell.
2. `scripts/check-no-dynamic-load.sh` source-level scanner (a companion to the archive-level no-dynamic-load leg already in the gate) and a `docs/console-portability.md` internal (gitignore-local) doc.
3. platform-support Part-6 (auxiliary evidence) anchor + forbidden-word gate over tracked console artifacts.
4. A host-target link probe (`cc probe.c libvokra.a`) — a stronger "it links" proof than nm presence; not needed for the symbol-presence contract, and musl has no linker on the build machine.

## (e) Non-negotiables carried forward

- **CPU-only for consoles.** The GPU backends (CUDA / Vulkan / and the future NPU delegates) are dlopen install-model backends; static-linking them would mean bundling their libraries, which collides with the dlopen model and zero-dep. Console GPU support, if ever, is a future SDK-specific backend — not this WP.
- **panic = unwind stays** on the shipped runtime profile (ffi_guard firewall).
- **No console model name / SDK name / target triple / SDK API name** appears in any tracked (public) artifact — this handoff, the CI YAML, and the platform-support matrix included.

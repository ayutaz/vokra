## 2. Seed the model-level RTF baselines (ubuntu-latest only)

The Silero / whisper-base / piper baselines ship as `$placeholder` files and
the gates clean-skip until seeded. **Seed from ubuntu-latest CI, never an M1
rig** (docs/bench-baselines/README.md — NEON skew false-fails the 5% gate).

1. Run `bench-baseline-capture.yml` (workflow_dispatch).
2. Download the `bench-baselines-ubuntu-latest` artifact; review the numbers.
3. Commit the reviewed `rtf` into the matching `docs/bench-baselines/*.json`,
   dropping the `$placeholder`. The whisper-base RTF is seeded from the
   `nightly-asr-wer` companion artifact instead (no extra download).

## 3. GitHub Pages (dashboard publish)

The dashboard artifact is produced on every `dashboard.yml` run. To publish it
as a page: enable Pages in repo settings (Settings → Pages → Source =
GitHub Actions), then set repo variable `VOKRA_PAGES_ENABLED=true`. Until
then the `deploy` job is skipped and only the artifact is produced.

## 5. Tier 2 device lab (self-hosted runners) + staged GGUFs

`nightly-tier2-device.yml` measures NFR-PF-09/10 on real hardware; the lab is
owner (§11 machine-acquisition). To activate:

1. Register Pi runners with labels `rpi5` / `rpi4` / `rpi3b` / `pizero2w`.
2. Set repo variable `VOKRA_TIER2_SELF_HOSTED=true`.
3. Stage the GGUFs on each runner and point the repo variables at them:
   `VOKRA_TIER2_WHISPER_BASE_GGUF`, `VOKRA_TIER2_WHISPER_TINY_GGUF` (Q4_K),
   `VOKRA_TIER2_SILERO_GGUF`, `VOKRA_TIER2_KOKORO_GGUF` (+ `VOKRA_TIER2_KOKORO_STYLE`,
   `VOKRA_TIER2_KOKORO_PHONEMES`), and `VOKRA_TIER2_WAV`.
4. Remove the `if: false` guard on the `measure` job (it exists so the cron
   cannot red with "no matching runner" before the fleet exists — see the

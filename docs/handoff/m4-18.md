## (e) 既知の制約（honest notes）

- CLI の happy path（実 GGUF での scoring）は **T05 converter 到着まで実行不能**（UTMOS GGUF を作る supported 経路が converter のみ。in-crate round-trip テストが from_gguf → score の全経路を検証済）。
- parity harness の synthesized 拒否は**構築来歴 flag**に基づく: 合成値を GGUF 経由で流し込めば flag は消える。実 reference の honesty は fixture `provenance` + README recipe + owner 運用が担う。
- v0 skeleton semantics（feature encoder 正規化なし / positional encoding なし / head 線形間非線形なし）は **upstream 一致主張ではない**。flip 時に v0 で表現不能なら variant bump。

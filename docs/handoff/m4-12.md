# M4-12 handoff: v1.0 GA ABI freeze preparation

- **Producer WP**: M3-16 (v0.9 ABI changelog scaffold — this file lands under M3-16-T05).
- **Consumer WP**: M4-12 (C ABI freeze + semver compliance start at v1.0 GA tag).
- **Requirement**: IF-01 (v1.0 以降 semver 準拠 = `docs/system-requirements.md` §5 IF-01 row), FR-API-01 (single header `include/vokra.h`), NFR-MT-03 (changelog automation).
- **Boundary (must not be misread)**: **This WP (M3-16) does NOT fire the freeze. The freeze fires in M4-12** at the v1.0 GA release tag (`docs/milestones.md` §8 M4-12 row, §7.2 M3-16 row = 2026-07-08 relabel from v0.9/M3 → v1.0 GA/M4-12).

## (a) Where the v0.9 changelog already lives

Both files are landed on `feat/m3-plan-and-wave1` and stay authoritative through the v1.0 tag:

- **`docs/abi-changelog.md`** — machine-checkable narrow ABI record (C symbols in `include/vokra.h` + GGUF `vokra.*` schema). Includes a `## Baseline snapshot: v0.9.0-dev` section, per-PR `## Entries`, and an existing `## Handoff to M4-12 (v1.0 GA freeze)` section that this file lifts into a standalone inbox.
- **`CHANGELOG.md`** — human-facing Keep-a-Changelog prose. `[Unreleased]` section covers v0.5/M2; v0.9/M3 additions accrue there through the v0.9 tag.

## (b) Work M4-12 must perform at the v1.0 GA tag

1. **Re-anchor**: copy the v1.0 `include/vokra.h` symbol list to `docs/abi/vokra.h.v1.0-baseline.symbols`; switch `scripts/check-abi-changelog.sh` to diff against that file (currently anchored at `docs/abi/vokra.h.v0.9-baseline.symbols`, see the `ANCHOR=` assignment near the top of the script).
2. **Declare the freeze in-header**: amend the `STABILITY` block at the top of `include/vokra.h` from "the ABI is NOT frozen" to the IF-01 freeze commitment (`docs/adr/0003-c-abi-design.md` §決定 / IF-01 clause is the wording anchor; `docs/system-requirements.md` §5 IF-01 row is the requirement wording).
3. **Release-note wording**: state in the v1.0 GA release notes that (i) `include/vokra.h` and the `vokra.*` GGUF schema are frozen under IF-01 and (ii) every post-v1.0 breaking change requires a semver major bump. Roll v0.9 entries in `docs/abi-changelog.md` into a "0.9 → 1.0 delta" summary under `CHANGELOG.md` `[1.0.0]`.
4. **Clear the prerelease window**: after the roll-up, empty the `## Entries` section of `docs/abi-changelog.md` for the next prerelease window (v1.1.x or v2.0.0-dev), keeping schema / policy / baseline-snapshot sections intact.

## (c) Post-v1.0 GA breaking-change rule

- Post-v1.0 GA, **breaking changes to `include/vokra.h` or the `vokra.*` GGUF schema require a major version bump (v2.0.0)** — no exceptions during minor/patch releases. Prerelease permissiveness (v0.9.x free rename/remove) ends at the v1.0 tag; `docs/abi-changelog.md` §`Pre-1.0 policy (prerelease semver)` already states this policy.
- Additions that are backward-compatible in a C-linkage sense (new function, new `#[non_exhaustive]` enum variant, new opaque handle type) remain permitted at minor bumps.

## (d) CI gate candidate to promote in M4-12

- Tool: **`scripts/abi-diff.sh`** — pure-bash anchor differ produced by M3-16-T02. It is deliberately advisory today (script's own `NOT WIRED INTO CI` block: "Per M3-16 spec §T02 last paragraph: CI gating is left to M4-12"). M4-12 promotes it by (i) re-anchoring against `docs/abi/vokra.h.v1.0-baseline.symbols` (see (b) step 1) and (ii) failing the CI job if any delta appears without a `CHANGELOG.md` `[1.x.y]` entry + a semver-major bump for any breaking delta.
- Companion advisory-→-required flip: `scripts/check-abi-changelog.sh` (currently the per-PR advisory changelog gate). Post-freeze it becomes a required check that blocks merge on `main` (`docs/abi-changelog.md` §`Handoff to M4-12 (v1.0 GA freeze)` — see the `Promote scripts/check-abi-changelog.sh from advisory (M3-16) to required CI check` bullet). M3-16 ships tool + baseline; M4-12 owns the `.github/workflows/ci.yml` wiring + GitHub branch protection update.

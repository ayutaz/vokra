## (c) 新規発見バグ / gap 一覧

### P1 — open 4 件（+ patch 済み 2 件）

1. **[wt-whisper] Whisper word-timestamp の off-by-one row slice**: `crates/vokra-models/src/whisper/beam_glue.rs` `whisper_word_timings` が `text_lo = n_prefix`（decoder_start_ids 4 個 = `<|notimestamps|>` を**含む**）で切るのに対し、openai timing.py:215 は `len(sot_sequence)`（3 個 = notimestamps を**除く**）で切る → 先頭 text token の emission 行が落ち、全 word の start/end が 1 emission 行遅れる（生 Δ mean 212–443 ms、診断 \|v.start−r.end\| は mean 30–69 ms に収束 = DTW core は一致）。**修正案は特定済**: `text_lo = n_prefix - 1`、n+1 行を `[c_0..c_{n-1}, eot]` として扱い、eot 行到達を最終語 end に使う（下記 P2 の 30 s leak も同時解消）。
2. **[server-real] Wyoming 実 client 非互換**: `integrations/vokra-server/src/api/wyoming.rs:645-653` `pump_events` が inbound event の `data` continuation を読んだ後**破棄**（header.data へ merge しない）。upstream wyoming **>=1.2.0（現行 1.10.0 まで、pip source bisect 1.0.0/1.2.0/1.4.0/1.5.2/1.10.0 で確定）= HA 世代 client 全て**は data を continuation でのみ送るため、describe/info 以外の**全推論が失敗**（audio-start/chunk "missing data"、transcribe "no audio received"、synthesize は text 消失）。genuine wyoming 1.10.0 package で live 再現（`logs/wyoming-upstream-framing.json`）。repo 自身の wyoming-design.md §3（data_buf を dispatch へ渡す）とも矛盾。修正 = continuation merge + repo tests の framing（現状 inline-only = <=1.1.x 相当）修正。
3. **[server-real] production TTS が plain-text 不能**: `server.rs build_service → InferenceService::build` が `PassthroughPhonemizer` 固定で `with_phonemizer`（実 8 言語 G2P 注入）を main path が未使用 → 全 surface で `PassthroughPhonemizer: 'Hello' is not a phoneme id` の明示エラー。raw phoneme-id 入力は e2e 動作（1.196 s 非無音、決定的 4/4）。campaign-1 の CLI --text placeholder gap の server 版（より厳格な hard-error 形）。
4. **[dfn3-real] denoise op が実 DFN3 を載せられない**: M4-20(c) の実 checkpoint 133 tensor（2.17M params、conv+BN+GroupedLinear+GRU + 学習済 filterbank）に対し vokra は 6-tensor / 263k dense scaffold で **name overlap 0**、converter に ckpt parser なし。載る唯一の重み（synthetic）では音声を noisy より 34 dB 劣化させる（SI-SNR -29.169 dB）。v1.0-rc は「DeepFilterNet (MIT) 統合」の claim の裏で untrained topology shell を出荷している状態。**T17 = op-level conv+GRU+filterbank 実装が必要（converter fix では閉じない）**。ground-truth tensor 一覧（ckpt-tensors.tsv）は納品済。

**P1 FIXED（mimi-roundtrip.patch 内、要 verify-on-HEAD）**: (i) Mimi PCM roundtrip 未配線（converter が vokra.mimi.\* config も structural tensor も emit せず — campaign-1 エラーを verbatim 再現後、T29 設計どおり閉鎖）、(ii) encoder RVQ semantics 乖離（単一 input_proj + 32 深 chain vs upstream SplitResidualVectorQuantizer の **2 並列 projection**（semantic 1 深 + acoustic 31 深）— 名前が繋がっても acoustic codes が誤る問題を presence 駆動で修正、旧挙動は chunk 不在時に保存）。

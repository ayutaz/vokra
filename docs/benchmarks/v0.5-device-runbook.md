### 2.1 実行手順 (M2-02-T13 handover doc 参照)

以下の順に [`docs/m2-14-ios-rtf-handover.md`](../m2-14-ios-rtf-handover.md) を通す:

1. § 1 **Prerequisites checklist** — XCFramework build、Package.swift、Xcode 14+、`whisper-base.gguf`、`jfk-30s.wav` (§1.3 と同一 fixture)、iOS 15+ 実機、Developer signing profile。**動的ロード禁止 (I1)** / **JIT 不使用 (I2)** の遵守は既に XCFramework build 時に保証されるが、本 runbook § 0.2 で再確認する。

4. § 4 **Recording template** — CPU / Metal 両バックエンド行、Device model、iOS version、Elapsed、RTF、NFR-PF-03 判定。**この記入は [`docs/benchmarks/v0.5-device-benchmarks.md`](v0.5-device-benchmarks.md) § 1 表に転記する** (M2-14-T05、handover § 4 は raw 記録、公開ページは公開値)。
5. § 5 **Backend selection** — `vokra_session_set_backend(s, VOKRA_BACKEND_METAL)` で Metal に切替。**FR-EX-08 (I3): unsupported op は explicit error として返る、silent CPU fallback しない**。rc をログに残す (fabricated pass 禁止)。
6. § 6 **R4 boundary — Metal probe failure on iPhone** — Metal init が iPhone で失敗 (例: `MTLGPUFamily.Apple7` 未認識、M2-01 の device probe が macOS `MTLGPUFamily.Mac*` 前提で書かれていた場合) は、**本 runbook では patch せず**:
   - CPU-only RTF は測れる → 記録する
   - Metal 行は `blocked-by-M2-01` として記録
   - M2-01 に defect ticket (XCFramework SHA256、iPhone model + iOS version、`NSError` from Metal init、`MTLDevice.supportsFamily(_:)` 結果)
   - M2-01 が iOS GPU-family fix を出荷したら本 runbook を再実行

## 3. CUDA Whisper large-v3 RTF 計測 (NFR-PF-04 < 0.1)

### 3.1 事前確認: NVIDIA EULA 5 点遵守 (M2-14-T06 / U1〜U4)

- [ ] **(1) cudart / cudnn / cublas 非 bundle** — CI job `unity-package-nvidia-check` (`scripts/check-unity-package-no-nvidia.sh` 起動) が green、または `nm libvokra.so | grep '__cud'` が empty。
- [ ] **(2) dlopen 検出** — `LD_DEBUG=libs vokra-cli bench --backend cuda 2>&1 | grep -E 'libcuda\.so\.1|libnvrtc\.so\.12'` に両者が現れる (2026-07-08 vast.ai run で確認済 = [`docs/m2-cuda-rtf-variance-2026-07-08.md`](../m2-cuda-rtf-variance-2026-07-08.md) § Silent CPU fallback ruled out の LD_DEBUG 行と同じパターン)。

- [ ] **(4) cuDNN 必須依存でない** — CUDA backend crate の linkage に cuDNN が含まれず、cuDNN 不在 host で backend が起動できる (M2-03-T06)。

```bash
# Path A: decomposed 2 + 7·n_head chain (VOKRA_CUDA_DISABLE_FA_V2=1)
./tools/parity/cuda_rtf_variance.sh \
    --gguf   /root/whisper-large-v3.gguf \
    --audio  /root/jfk-30s.wav \
    --iters  10 --warmup 1 \
    --fa-v2  off --label decomposed \
    --output /root/rtf-decomposed.jsonl

# Path B: default FA v2 gated wrapper (t_q >= 16)
./tools/parity/cuda_rtf_variance.sh \
    --gguf   /root/whisper-large-v3.gguf \
    --audio  /root/jfk-30s.wav \
    --iters  10 --warmup 1 \
    --fa-v2  on --label gated_fa_v2 \
    --output /root/rtf-fa-v2.jsonl

# Analyzer: JSONL → markdown report (mean/median/CV/p50/p95/p99/histogram)
./tools/parity/cuda_rtf_analyze.py /root/rtf-decomposed.jsonl \
    --output /root/rtf-decomposed.report.md
./tools/parity/cuda_rtf_analyze.py /root/rtf-fa-v2.jsonl \
    --output /root/rtf-fa-v2.report.md
```

**boundary** (U3 / U6):

- decomposed path (`--fa-v2 off`) は `VOKRA_CUDA_DISABLE_FA_V2=1` で attention を分解 (GEMM + softmax + GEMM) 実行 = M2-03 baseline。
- gated FA v2 path (`--fa-v2 on`) は `vokra_flash_attn_v2_causal_f32` を `t_q >= 16` gating で呼ぶ。**Whisper decoder-step は `t_q = 1` なので hot path は decomposed path に fall through** (silent CPU fallback ではない、GPU 内での fall through、FR-EX-08 保存、[`docs/adr/M2-03-followup-rtf.md`](../adr/M2-03-followup-rtf.md) §D9)。
- **shared memory < 40 KB / block の device では FA v2 = `use_flash_attn = false` に自動 disable** される (`log::info!` で明示、silent ではない、[`docs/adr/M2-03-followup-rtf.md`](../adr/M2-03-followup-rtf.md) §D9)。
- **`--force-flash-attn` は使わない** — 本 runbook は default gating の測定に徹する (probe 不足時に明示エラーで止まる `--force-flash-attn` は debug 用途、計測 run では使わない)。

**analyzer が emit する CV verdict** (`cuda_rtf_analyze.py`):

- `OK` — CV = stddev/mean <= 0.20 (analyzer threshold、これは **formal `<0.10` gate ではない**、`tools/parity/README-cuda-rtf-variance.md` § Zero-dep 節と整合)。
- `WARNING` — CV > 0.20 (hardware variance が大きい)。この場合は追加 iter や別 offer での再測定を検討 (§5 の 2026-07-08 vast.ai run のように systemic host slowdown なら追加 iter は無効)。
- **analyzer は RTF ceiling を assert しない** ([`docs/adr/M2-03-followup-rtf.md`](../adr/M2-03-followup-rtf.md) §D6 red-line)。

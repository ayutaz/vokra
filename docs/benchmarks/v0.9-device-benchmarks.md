## 6. vast.ai spot RTX 4090 CC-side reference (2026-07-11 UTC)

**Position in plan**: **reference measurement only** — NOT the formal always-on gate (M2-14 self-hosted runner + M3-01 5% regression、`docs/adr/M2-03-followup-rtf.md` §D6)。CC 側の workflow (`tools/parity/vast-e2e-measurement.sh`) で採取した vast.ai spot RTX 4090 の 1 回きり参考値。実機 owner lab / X-06 nightly matrix の代替ではない。

### 6.1 M3-01 CUDA Whisper large-v3 RTF variance (N=10)

**Environment**:
- **GPU**: NVIDIA GeForce RTX 4090 (vast.ai spot、offer 42160295、driver 560.35.03)
- **GGUF SHA-256**: `2ebfc46a95ad3831377ae5f4d9d30e35dd2d87fb0526769a02f78b237d30e761` (M2 baseline `docs/bench-baselines/whisper_large_v3_cuda_rtf.json` と bit-identical)
- **Audio**: `tests/fixtures/audio/jfk-30s.wav` (harness reads audio_seconds = **11.0** = 実音声長、pad は除外)
- **Toolchain**: rustup 1.88 stable (let_chains 対応必須)
- **branch / SHA**: `feat/m3-plan-and-wave1` @ `7ed5a30`（PR #4 で main に squash-merge 済、post-merge = `1f934da` @ main、2026-07-11）

| Arm | mean | median | p50 | p95 | p99 | min | max | stddev | CV | Verdict (vs < 0.10 gate) |
|-----|-----:|-------:|----:|----:|----:|----:|----:|-------:|---:|:-----------------------:|
| decomposed (FA_V2 off) | 0.766 | 0.776 | 0.776 | 0.787 | 0.787 | 0.736 | 0.787 | 0.019 | **0.024** | FAIL |
| gated_fa_v2 (FA_V2 on) | 0.782 | 0.780 | 0.779 | 0.821 | 0.821 | 0.753 | 0.821 | 0.019 | **0.024** | FAIL |

**Interpretation (owner 判断のための素データ)**:
- CV = 0.024 (< 0.20 gate) = 計測は stable、median は信頼できる。M2 baseline `whisper_large_v3_cuda_rtf.json` の stddev 0.00016 (CV ≈ 0.0014) には及ばず = spot instance ゆえの hardware variance の反映。
- **audio_seconds が 30 → 11 に変わったため M2 baseline 直接比較不可**: M2 は synthetic 30 s tone (RTF 0.113 median)、本測定は `jfk-30s.wav` の実音声 11 s + 19 s zero-pad ゆえ harness は audio_seconds = 11 で RTF を計算。RTF 分母が ~2.7x 小さい = RTF ~2.7x 大きく出る想定。0.113 × 2.7 = 0.31 と比較しても実測 0.77 は **~2.5x 上振れ = 実 regression が存在**。
- **FA v2 arm が decomposed より遅い**: FA v2 gate `FA_V2_MIN_TQ = 16` が decoder-step (t_q = 1) では発火しない = 実質「decomposed + gate 判定 overhead」。FA v2 の性能上乗せは encoder 経路のみで decoder は decomposed 相当。
- **判定**: **どちらも NFR-PF-04 の < 0.10 always-on 目標に大きく未達** (0.77 vs 0.10)。原因分析は M3-01 (weight caching / inter-head overlap / session pool / FA v2 gate 見直し) or M2-14 self-hosted runner での再計測 (spot vs dedicated の delta 切り分け) が follow-up。

**Raw artifacts**:
- `docs/bench-baselines/vast-2026-07-10/rtf-decomposed.jsonl` + `.report.md` (10 iter per-sample + histogram)
- `docs/bench-baselines/vast-2026-07-10/rtf-fa-v2.jsonl` + `.report.md`

### 6.2 M3-15 in-process reference bench (FakeSynth floor)

**Environment**: 同 vast.ai RTX 4090 instance で `integrations/vokra-server/benches/tts_latency.rs` を実行。**FakeSynth 経路ゆえ engine backend 独立の schema decode + dispatch + WAV encode オーバーヘッドの floor 値** (実 Metal / CUDA engine ではない)。

| Boundary | median (µs) | p50 (µs) | p95 (µs) | max (µs) | Budget (ms) | Verdict |
|----------|------------:|---------:|---------:|---------:|------------:|:-------:|
| http_end_to_end     | 87 | 87 | 178 | 207 | 90 (v0.5) / 75 (v0.9) | **PASS** (floor << 75 ms) |
| time_to_first_audio | 34 | 34 | 36  | 55  | 90 (v0.5) / 75 (v0.9) | **PASS** (floor << 75 ms) |

**Interpretation**:
- FakeSynth floor は 87 µs = **0.087 ms** = 75 ms budget の **1/860**。schema layer + dispatch layer overhead は 75 ms budget を easily 満たす。
- **実 engine wire (Metal / CUDA) の TTFA は本測定の対象外**: piper voice GGUF + running vokra-server + `vokra-server-bench` HTTP client の 3 点セット必須。owner supplied piper voice GGUF 到着後に owner-side で `docs/m3-15-server-latency-handover.md` § 4 経路 C (`vokra-server-bench`) で採測。
- **NFR-PF-05 < 75 ms 実測 always-on gate** は X-06 nightly self-hosted matrix + 依頼者 lab の一回性判定に依存 = 本 measurement は floor 提示のみで gate promotion しない。

**Raw artifact**: `docs/bench-baselines/vast-2026-07-10/m3-15-in-process.txt`

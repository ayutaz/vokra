## 2. CUDA Whisper large-v3 RTF (NFR-PF-04 < 0.1)

**閾値**: NFR-PF-04 = Whisper large-v3 CUDA 実機 RTF < 0.1 (v0.5 値、`docs/system-requirements.md` L294)。
**判定バックエンド境界** (FR-EX-08 / NFR-RL-06): CUDA 非互換 (driver / CUDA / cuDNN バージョン、GPU 非搭載) は `vokra_cuda_probe()` が **silent fallback せず明示エラー** (M2-03-T05/T06、CLAUDE.md M2 status)。
**Backend 実装メモ** (提案の T01 spec の "cudarc + FA v2 CPU port" 記述は M2 実装で更新済): M2-03 CUDA backend は **cudarc 不使用の生 FFI** — Driver API `libcuda` + NVRTC を dlopen 実行時ロード (NVIDIA EULA 準拠 install モデル + zero-dep NFR-DS-02、CLAUDE.md M2 status)。**FA v2 は kernel 実装済** (`vokra_flash_attn_v2_causal_f32`、M2-03-followup-rtf ADR §D2)。**FA v3 (Hopper WGMMA/TMA) は v1.5+ = M4-07、前倒し禁止** (CLAUDE.md 設計制約 §5-(7))。
**EULA 5 点遵守記録**: (1) cudart/cudnn/cublas 非 bundle、(2) `dlopen("libcuda.so")` / `LoadLibrary("nvcuda.dll")` で system install 検出、(3) `vokra_cuda_probe()` 非互換=明示エラー、(4) cuDNN 非必須、(5) `third_party/NVIDIA-EULA.md` (M2-03-T02 / T23)。計測前に `vokra_cuda_probe()` 出力 (下記 GPU / driver / CUDA / cuDNN 列) を貼り付け、5 点の遵守を計測記録に残す (M2-14-T06)。

## 4. Escalation Policy

- **閾値未達値をそのまま記録・公開する** (新規緩和目標を発明しない、M2-14-T05/T08 red-line、M2-06-T14 の「M2-14 に委譲」を受ける最終判定)。
- 未達の起票先: iOS RTF → M2-01 (Metal) / M2-02 (iOS ビルド) / CUDA large-v3 RTF → M2-03 (CUDA) / M2-04 (fusion)。
- **M2-15 四半期 Go/No-go review** (NFR-MT-05、kill-switch 自動監視 .yml は 2026-07-04 依頼者決定で廃止 → 手動四半期 review に集約) に本ページ実測値を判定材料として持ち込む (M2-14-T10 § 「M2-15 引き渡し」)。addressable market 指標 (Kill switch K) とは別ゲート。
- **X-06 nightly ダッシュボード** への引き継ぎ (M2-14-T10 § 「X-06 引き継ぎ」): 本 v0.5 実機実測値を X-06 Tier 2 nightly 実機トラッキング (NFR-MT-02) の baseline として登録。M3-01 の 5% regression + hard ceiling 0.10 gate は本ページ実測値 (self-hosted CUDA runner 上での値) を baseline として発火する (`docs/perf/cuda-large-v3-baseline.json` の `gate_expression`)。
- **watermark / research flag 対象外**: 本ページの計測に用いる weight は Whisper base / large-v3 = **MIT only** (公式 model zoo 搭載、M2-06-T16)。CC-BY-NC 系 weight (F5-TTS / Fish-Speech / EnCodec) は使用しない = compliance gate (M2-13) 自動拒否対象、目視確認不要。ASR は音声→テキストのため watermark (FR-CP-01/02、TTS/VC 出力向け) は非該当 (M2-06-T19 の non-applicability 整理、CLAUDE.md M1-07 ドロップの読み替え)。
- **FR-EX-08 non-viable path**: 本ページには「実際に走ったバックエンド」の値のみが記録される。silent CPU fallback を許容した数値の掲載は本 escalation policy 違反 = 記録は許されるが Verdict は必ず `FAIL` として扱い、fallback を作った PR を revert 対象化する (M2-01 / M2-03 の完了条件)。

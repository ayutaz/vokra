# Vokra Unity demo (VAD ‚Üí ASR ‚Üí TTS)

A single-scene Unity demo that runs **Silero VAD ‚Üí Whisper base ASR ‚Üí
piper-plus native TTS** entirely through the Vokra C ABI (`include/vokra.h`,
M0-09). It is the M0-10 work package deliverable (v0.1 spike).

Inference goes **only** through Vokra ‚Äî there is no Unity Sentis / Inference
Engine dependency (`Packages/manifest.json` has none; SRS ¬ß5-(10),
CLAUDE.md design note 5) ‚Äî and the end-to-end path contains **no onnxruntime**
(the TTS is the piper-plus native MB-iSTFT-VITS2 implementation, 2026-07-02
native decision).

## Scope boundaries (M0-10-T01)

This demo is intentionally narrow. Out-of-scope items and where they live:

- **(a) Desktop 3 OS only** ‚Äî macOS / Linux / Windows. **iOS**
  (`DllImport("__Internal")` static linking, **NFR-RL-03**) and **Android**
  (`StreamingAssets` jar-URL ‚Üí `persistentDataPath` extraction, **NFR-RL-04**)
  are the **v0.5 official plugin** (**FR-API-04**: demo = v0.1 spike, official
  plugin = v0.5).
- **(b) Single worker thread** ‚Äî `Session`/`Stream` thread-safety
  (`Send + Sync`, atomic ref count, **FR-API-03**) is **v0.1 MVP** scope, so the
  demo makes every C ABI call on one worker thread (`PipelineRunner`) and never
  relies on runtime thread safety.
- **(c) No resampling** ‚Äî `resample` (**FR-OP-04**) is **v0.1 MVP** (milestones
  M1-06). Input WAV must already be at the model's rate (16 kHz mono); a mismatch
  is an explicit error. `soxr`/`rubberband` (GPL) are excluded as a design red
  line regardless.
- **(d) Inference via Vokra C ABI only** ‚Äî no Unity ML package is used
  (**SRS ¬ß5-(10)**).

## Setup and run (all 3 OS, same steps)

1. **Unity version** ‚Äî open this folder in your Unity **LTS** editor; it writes
   `ProjectSettings/` (see `ProjectSettings/README.md`). Mono backend is fine;
   the callback design is IL2CPP-safe (NFR-RL-02).
2. **Native library** ‚Äî from the repo root, on each OS:
   ```sh
   bash scripts/build-unity-plugin.sh   # cargo build -p vokra-capi + copy into Assets/Plugins
   ```
   (Windows without Rust: use the M0-01 CI Windows `vokra.dll` ‚Äî see
   `Assets/Plugins/README.md`.)
3. **Models + test audio**:
   ```sh
   bash scripts/fetch-demo-models.sh    # VAD fixture + test_16k.wav; ASR/TTS via env or vokra-convert
   ```
   VAD works immediately from the committed fixture; ASR/TTS activate once
   `whisper-base.gguf` / `voice.gguf` are placed (see
   `Assets/StreamingAssets/models/README.md`). Set `VOKRA_WHISPER_GGUF` /
   `VOKRA_PIPER_GGUF` before step 3 to stage them automatically.
4. **Run** ‚Äî open `Assets/Scenes/VokraDemo.unity`, press Play, click **Run
   pipeline**; the log shows the VAD frame count, ASR text, and TTS output, and
   **Play TTS** plays the synthesized audio.

> If the committed scene fails to open in your Unity version, recreate it in ~30 s:
> *File ‚ñ∏ New Scene* ‚Üí add an empty GameObject ‚Üí *Add Component ‚ñ∏ Demo Ui* ‚Üí save
> as `Assets/Scenes/VokraDemo.unity`. All logic is in the `DemoUi` component.

### Headless mode (machine-checkable, used for Linux ‚Äî T11)

The player runs the pipeline once with no UI when launched with `-vokraInput`
(or in `-batchmode`), then quits with an exit code (0 = ok, 1 = a stage errored):

```sh
./VokraDemo -batchmode -nographics \
  -vokraModelsDir /path/to/models \
  -vokraInput  /path/to/test_16k.wav \
  -vokraOutput /path/to/tts_out.wav
```

It logs each stage (`[vokra-demo][Vad]/[Asr]/[Tts]`) and writes the TTS WAV to
`-vokraOutput`. No audio device or display needed. `-vokraText <text>` overrides
the sentence spoken when ASR has no real text yet.

## Design notes

- **UI = IMGUI (`OnGUI`).** The ticket proposed uGUI; this demo uses IMGUI so the
  whole UI is code-only (no Canvas/EventSystem/Font/TMP setup that varies by Unity
  version and needs scene authoring). The v0.5 official plugin can ship a richer
  uGUI/UI-Toolkit front end.
- **Callbacks = polling in M0.** `include/vokra.h` exposes **no** callback API;
  streaming is poll-based (`vokra_stream_push_pcm` + `vokra_stream_poll`). So
  `VokraCallbacks.cs` is a compile-checked **template** of the IL2CPP-safe pattern
  (`[MonoPInvokeCallback]` static method + `GCHandle` userdata, NFR-RL-02),
  carried to v0.5 (FR-API-04) ‚Äî it is not wired to any current symbol.
- **ASR text in M0** renders as bracketed token ids (the tokenizer is not yet
  embedded in the GGUF ‚Äî an M0-09 followup); the demo speaks a fallback sentence
  in that case.

## vokra.h ‚Üî C# mapping

The DllImport declarations in `Assets/Scripts/Vokra/NativeMethods.cs` mirror
`include/vokra.h` one-to-one (session create/destroy, `asr_transcribe`,
`tts_synthesize`, `stream_open`/`push_pcm`/`poll`/`destroy`, `string_free`,
`audio_free`, `last_error`, `version`). The header is the source of truth; the
per-symbol table is in that file's header comment.

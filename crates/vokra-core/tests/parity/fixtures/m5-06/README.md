# M5-06 `wfst_decode` parity fixtures

Committed fixtures for `crates/vokra-core/tests/parity_wfst.rs` (T14). They are
generated by **real OpenFST** (`tools/parity/wfst_dump_reference.py`, T13a/T13b)
— the independent oracle. A self-computed reference would verify nothing
(numerical-parity discipline), so this test's reference *must* come from
OpenFST. Because the fixtures are committed, the Rust test needs no OpenFST and
CI reads the committed reference.

## Files (per fixture `linear` / `epsilon` / `nbest`)

- `<name>.fst` — the decode graph as a **real OpenFST `VectorFst<StdArc>`
  binary** (`fstcompile --arc_type=standard --fst_type=vector`). This is the
  input to `read_openfst_vector` — verifying the from-scratch reader against a
  real producer.
- `<name>.expected` — text: the parsed graph structure (from OpenFST
  `fstprint`/`fstinfo`, for the reader check), the emission matrix fed to the
  decoder, and the reference best-path + n-best (from
  `shortestpath(compose(E, G))` / `--nshortest`).

## Fixtures

| name    | what it exercises                              | best path            | cost   |
| ------- | ---------------------------------------------- | -------------------- | ------ |
| linear  | plain emitting arcs, single path               | words `[10, 20]`     | 1.175  |
| epsilon | epsilon closure at the start **and** mid-path  | `[40, 10, 30, 20]`   | 1.245  |
| nbest   | two parallel, distinct-word, distinct-cost paths | 2-best               | 1.175 / 1.225 |

Each fixture is engineered so the optimum (and each n-best rank) is **strictly
unique** — parity never depends on tie-breaking (ADR M5-06 §6). Costs are
tropical (negative-log; lower is better). The emission matrix indexes acoustic
labels directly (`emission[frame][ilabel]`, index 0 = epsilon sentinel, never
consumed).

## Provenance / license (owner sign-off: M5-06-T17)

- The `.fst` binaries are **tiny synthetic FSTs authored in
  `tools/parity/wfst_dump_reference.py`** (3–5 states each), not derived from
  any external corpus or copyrighted dataset. They were compiled by
  **OpenFST 1.8.4** (Apache-2.0), used at dev time only — the Vokra runtime
  links no OpenFST code (from-scratch Rust reader/decoder). See
  `docs/license-audit.md` (OpenFST row) and `docs/handoff/m5-06.md`.
- These are `-filter -diff -merge` plain blobs (`.gitattributes`) — never Git
  LFS pointers, so the byte payload the reader parses is exact.

## Regenerating

```
brew install openfst    # Apache-2.0, bottled
python3 tools/parity/wfst_dump_reference.py
```

The dumper aborts loudly if the OpenFST CLI is missing (it never self-mirrors).
